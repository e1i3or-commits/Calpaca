# Security policy

Please report suspected vulnerabilities through
[GitHub private vulnerability reporting](https://github.com/e1i3or-commits/Calpaca/security/advisories/new).

Do not open a public issue for authentication bypasses, token exposure,
cross-tenant access, calendar data disclosure, injection, or other security
problems. Include the affected revision, reproduction steps, impact, and any
suggested mitigation. Remove unrelated personal data and credentials.

Calpaca is under active development. Security fixes target the current `main`
branch; older revisions may not receive patches.
